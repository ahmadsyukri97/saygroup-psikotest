Setup Supabase dari Nol

Ikuti urutan ini persis. Setiap langkah menunjukkan tepat di mana harus klik — tidak perlu menebak-nebak menu mana yang dimaksud.

1. Buat Project2. SQL Editor3. Jalankan schema.sql 4. Jalankan seed5. Buat Admin6. Ambil API Key7. config.js
1
Buat akun & project baru

Buka supabase.com → Sign up (bisa pakai akun GitHub) → tombol New project. Isi nama project bebas (mis. "saygroup-assessment"), buat password database (simpan baik-baik), pilih region terdekat (mis. Singapore), lalu klik Create new project.

3
Klik "New query", tempel isi schema.sql, lalu Run

Di halaman SQL Editor, klik tombol + New query di kiri atas. Buka file supabase/schema.sql dari proyek yang saya kirim, salin seluruh isinya, tempel ke kotak editor, lalu klik tombol hijau Run (atau tekan Ctrl/Cmd + Enter).

4
Ulangi untuk seed-questions.sql

Klik + New query lagi (jangan ditumpuk di query yang sama). Salin seluruh isi supabase/seed-questions.sql, tempel, klik Run. Ini akan mengisi 60 soal starter ke tabel questions.

Aman dijalankan berkali-kali — query ini pakai on conflict ... do update, jadi tidak akan membuat soal dobel.
5

Di sidebar kiri, klik Authentication → tab Users → tombol Add user → pilih Create new user. Isi email & password untuk login dashboard HR nanti.

6
Jadikan user itu admin

Balik lagi ke SQL Editor → New query → jalankan perintah ini (ganti email sesuai user yang baru dibuat):

-- ganti email di bawah sesuai akun admin yang baru dibuat
insert into public.admins (user_id, email)
select id, email from auth.users where email = 'hr@perusahaan-anda.com';
Kalau berhasil akan muncul "Success. 1 row returned" atau serupa. User ini sekarang bisa login lewat halaman "Login Admin" di aplikasi.
7
Ambil Project URL & anon key

Sidebar kiri → Project Settings (ikon gerigi, biasanya paling bawah) → API. Salin nilai Project URL dan anon public key.

8
Tempelkan ke js/config.js

Buka file js/config.js di proyekmu, ganti dua baris ini dengan nilai dari Step 7:

// js/config.js
const SUPABASE_URL = "https://abcxyzproject.supabase.co";
const SUPABASE_ANON_KEY = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...";
Simpan, lalu buka index.html lewat local server (mis. npx serve .) atau setelah di-push ke GitHub Pages — halaman "Mulai Tes" sudah bisa memuat 60 soal dari Supabase.

Kalau masih error — cek di sini

permission denied for table questions
Berarti schema.sql belum sepenuhnya berhasil dijalankan (bagian Row Level Security-nya). Jalankan ulang seluruh isi schema.sql dari awal sampai akhir dalam satu query, jangan sepotong-sepotong.
relation "public.questions" does not exist
seed-questions.sql dijalankan sebelum schema.sql. Jalankan schema.sql dulu sampai sukses, baru seed-questions.sql.
syntax error at or near "..."
Biasanya karena hanya sebagian isi file yang ke-copy. Buka file aslinya, tekan Ctrl/Cmd+A lalu Ctrl/Cmd+C untuk pastikan semua baris tersalin, lalu tempel ulang (isi editor lama dihapus dulu).
Akun ini belum terdaftar sebagai admin
Muncul saat login di aplikasi. Artinya Step 6 (insert ke tabel admins) belum dijalankan, atau email yang dipakai beda dengan email user di Step 5. Cek dengan: select * from public.admins; di SQL Editor.
Gagal memuat soal. Cek koneksi atau konfigurasi Supabase.
Muncul di halaman tes. Biasanya js/config.js belum diisi / salah salin URL atau anon key. Bandingkan lagi persis dengan Step 7–8, tanpa spasi atau tanda kutip yang terpotong.
